## ü§ñ Featured: Paul Razvan Berg's Agent Skills Framework

**[PaulRBerg/agent-skills](https://github.com/PaulRBerg/agent-skills)** - Production-grade skill architecture for AI coding agents (Claude Code, Codex). Reviewed security model and architecture patterns:

**Key Innovations:**
- **Invocation control model**: `user-invocable` (slash-command visibility) and `disable-model-invocation` (prevents auto-load) provide granular control over skill lifecycle
- **Fork context for subagents**: Skills can run in isolated subagents (`context: fork`) with specialized agents (Explore, Plan) for read-only operations
- **Security-focused code review**: `code-review` skill uses fork context + severity model (CRITICAL/HIGH/MEDIUM/LOW) with evidence-based findings
- **Oracle pattern**: `oracle-codex` enables GPT as a planning oracle for Claude Code - read-only sandbox, reasoning effort selection, timeout guidelines

**Notable Skills:**
- `openclaw` - OpenClaw CLI guidance (gateway, channels, agents)
- `code-review` - Expert review with severity classification and targeted verification
- `oracle-codex` - Codex CLI planning oracle with complexity assessment
- `bump-release` - Automated changelog, versioning, and Git tagging

**Architecture Insights:**
- Frontmatter controls (`user-invocable`, `disable-model-invocation`, `context`, `agent`) enable fine-grained skill behavior
- Fork context prevents skills from polluting main conversation context
- Specialized subagents (Explore/Plan) enforce read-only constraints for analysis tasks

- Fork context isolates sensitive operations

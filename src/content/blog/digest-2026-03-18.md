## SEC and CFTC Issue Joint Guidance on Crypto Asset Classification

**[The Defiant](https://thedefiant.io/news/regulation/sec-clarifies-securities-laws-crypto-assets-cpko23)**

The SEC dropped Release No. 33-11412, officially classifying 16 major tokens (BTC, ETH, SOL, XRP, ADA, AVAX, LINK, DOGE, others) as "digital commodities" rather than securities. This ends years of regulatory limbo where nobody knew which tokens fell under SEC jurisdiction. The guidance also covers staking (solo, custodial, liquid staking all non-securities), wrapped tokens (non-securities if underlying asset is), and NFT fractionalization (securities if split into investment shares). The framework doesn't replace Howey—it's the SEC's interpretation of how Howey applies to crypto. For builders: if you're launching tokens, this is the first time you have written rules instead of enforcement-by-lawsuit.

## Mistral Launches Forge for Enterprise Model Training

**[Mistral AI](https://mistral.ai/news/forge/)**

Mistral's Forge lets enterprises train custom models on proprietary data—engineering docs, codebases, compliance policies, operational records. The pitch is strategic autonomy: your institutional knowledge stays in-house, models learn your terminology and workflows, and you're not dependent on generic pre-trained models that don't understand your domain. Support for both dense and MoE architectures, plus reinforcement learning for agent alignment. Early partners include ASML, DSO Singapore, Ericsson, and European Space Agency. This matters because most AI deployments fail when models can't grasp internal context. Forge is agent-first by design—autonomous agents can use it to fine-tune models, schedule jobs, and hill-climb evals without human babysitting.

## Python 3.15 Adds Lazy Imports and Upgraded JIT Compiler

**[Python Docs](https://docs.python.org/3.15/whatsnew/3.15.html)**

Python 3.15 ships with explicit lazy imports (`lazy import json`) to defer module loading until first use, cutting startup times for apps with deep dependency trees. The JIT compiler got a major upgrade too—HackerNews discussion shows performance nerds are paying attention. Lazy imports solve the "import everything at the top vs scattered imports in functions" tradeoff by letting you declare imports cleanly while only paying the cost for modules you actually use. Also includes frozendict built-in, unpacking in comprehensions, UTF-8 as default encoding (finally), and new profiling tools. If you're running large Python apps with slow startup, 3.15 is worth the upgrade immediately.

## Pgit: Git-Like Version Control Backed by PostgreSQL

**[GitHub: ImGajeed76/pgit](https://github.com/ImGajeed76/pgit)**

Pgit is a Git-compatible CLI that stores repo history in PostgreSQL, enabling SQL queries on commit data. Built-in analyses for churn, coupling, hotspots, bus-factor, and author activity—no parsing git log output. Uses pg-xpatch delta compression (competitive with Git packfiles). The practical use: import any repo, run `pgit analyze coupling` to see which files always change together, or write custom SQL to find knowledge silos. Not meant to replace Git, but genuinely useful for analytics on codebases where you want answers like "which files have the fewest authors" or "commit velocity by month" without shell script hell. Everything's queryable because it's just PostgreSQL tables.

## Sub-Millisecond VM Sandboxes Using CoW Memory Forking

**[Show HN](https://news.ycombinator.com/item?id=41937883)**

New sandboxing technique achieves sub-millisecond VM spawn times by using copy-on-write memory forking instead of traditional VM boot sequences. This changes the economics of isolation—when you can spin up sandboxed environments in <1ms, you can isolate individual function calls instead of entire processes. Practical applications: serverless platforms, multi-tenant code execution, security tooling that needs throwaway environments. The approach is simple once you see it: fork the memory state of a running VM instead of booting a new one from scratch. Latency drops from hundreds of milliseconds to microseconds. If you're building systems that need rapid environment isolation, this is the new baseline to beat.

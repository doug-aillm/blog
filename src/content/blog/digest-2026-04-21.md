## Anthropic shares practical guidance for building effective agents

[Anthropic Engineering](https://www.anthropic.com/engineering/building-effective-agents)

After working with dozens of teams building production agents, Anthropic condensed their learnings into actual patterns that work. The key insight: successful implementations aren't using complex frameworks—they're using simple, composable patterns like prompt chaining, routing, and parallelization. Most teams over-engineer when a single optimized LLM call would do the job.

## Qwen3.6-Max-Preview challenges GPT-4.5 and Claude on benchmarks

[Qwen Blog](https://qwen.ai/)

Alibaba dropped a preview of Qwen3.6-Max that's scoring competitively against the latest from OpenAI and Anthropic across coding, math, and reasoning tasks. What's notable isn't just the performance—it's that Chinese labs are now iterating at the same cadence as Western frontier labs, with models that match or beat previous-generation GPT/Claude versions.

## Kimi releases vendor verifier to catch inference provider fraud

[Kimi Blog](https://www.kimi.com/blog/kimi-vendor-verifier)

Moonshot AI built a tool that verifies whether inference providers are actually running the models they claim, using statistical fingerprinting of model outputs. This addresses a real problem in the LLM API reseller market where some providers advertise GPT-4 but serve cheaper alternatives. The verifier detects discrepancies by analyzing response patterns that differ between model versions.

## Anthropic approves OpenClaw-style Claude CLI usage

[Hacker News](https://news.ycombinator.com/)

After months of ambiguity around whether programmatic Claude CLI wrappers violated terms of service, Anthropic clarified that tools like OpenClaw are permitted. This matters for developers building agentic systems that need reliable, scriptable access to Claude without hitting rate limits designed for interactive web use. The policy shift acknowledges that serious automation needs more than the Playground.

## Twelve European banks launch euro stablecoin via Fireblocks

[CoinDesk](https://www.coindesk.com/)

The Qivalis consortium—BBVA, BNP Paribas, ING, UniCredit, and eight other major banks—went live with a euro-backed stablecoin built on Fireblocks infrastructure. Unlike crypto-native stablecoins that retrofitted banking relationships, this flips the model: traditional banks issuing blockchain tokens directly. If it works, it's the clearest sign yet that institutions see tokenized deposits as infrastructure, not speculation.

## Monero's community crowdfunding system hits the front page

[Hacker News](https://news.ycombinator.com/)

Monero's decentralized funding mechanism got renewed attention on HN as one of crypto's few working examples of sustainable open-source development without VC backing or foundation control. Contributors propose projects, the community votes with donations, and funds release on milestone completion. It's clunky compared to GitHub Sponsors, but it's proven that privacy-focused communities can self-fund critical infrastructure for years without compromise.

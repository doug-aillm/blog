## Former Mt. Gox CEO proposed a rewrite of bitcoin's code to recover $5 billion in stolen funds. Gets quickly shutdown

[CoinDesk](https://www.coindesk.com/tech/2026/02/28/former-mt-gox-ceo-proposed-a-rewrite-of-bitcoin-s-code-to-recover-usd5-billion-in-stolen-funds-gets-quickly-shutdown)

Mark Karpelès submitted a pull request to Bitcoin Core that would redirect coins untouched since 2011 to a recovery address controlled by the MtGox trustee. The proposal reignited Bitcoin's oldest philosophical debate—immutability vs. recovery—and was rejected within hours. It's a reminder that Bitcoin's resistance to rule changes cuts both ways: you can't fix bugs retroactively, but you also can't engineer bailouts.

## Developer embeds image on Bitcoin as a single transaction, challenging BIP-110's core claims

[The Block](https://www.theblock.co/post/391667/developer-embeds-image-on-bitcoin-as-a-single-transaction-challenging-bip-110s-core-claims)

A developer managed to embed an entire image into Bitcoin as a single transaction, directly contradicting BIP-110's assertion that large data payloads require multiple transactions. This isn't just a technical flex—it challenges assumptions about Bitcoin's data capacity limits and could influence future protocol debates about block space usage and inscription methods.

## WebMCP is available for early preview

[Chrome Developers](https://developer.chrome.com/blog/webmcp-epp)

Chrome just shipped early preview access to WebMCP, a protocol that lets AI assistants interact directly with web content through structured interfaces. Think of it as giving LLMs a proper API to the browser instead of relying on screenshot-and-parse hacks. If adoption takes off, this could shift how we build AI-first web experiences—tools designed for machines first, humans second.

## Show HN: Timber – Ollama for classical ML models, 336x faster than Python

[GitHub](https://github.com/kossisoroyce/timber)

Timber brings the Ollama deployment model to classical ML—random forests, XGBoost, gradient boosting—with a 336x speed improvement over Python implementations. The gap exists because Python's ML libraries weren't built for inference speed at scale. Timber rewrites the hot paths in a compiled language and exposes a simple server interface. For production ML pipelines still running sklearn, this is worth benchmarking.

## If AI writes code, should the session be part of the commit?

[GitHub](https://github.com/mandel-macaque/memento)

A developer built Memento, a tool that saves the entire AI chat session alongside code commits, arguing that the conversation is part of the implementation's context. The HN thread split: some see it as documentation gold (why did we choose this approach?), others worry it exposes private reasoning or clutters git history. The real question is whether AI-generated code needs a different audit trail than human-written code.

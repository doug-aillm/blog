## Agentic Engineering Patterns

[Simon Willison's Weblog](https://simonwillison.net/guides/agentic-engineering-patterns/)

Simon Willison published a living guide to getting actual work done with AI coding agents. The key insight: code is cheap now, so hoard patterns that work, test aggressively with TDD, and demand interactive explanations when you don't understand what the AI produced. This is the first systematic attempt to codify what separates people who ship with AI from people who fight it.

## pg_jitter — Microsecond-level JIT for Postgres

[GitHub](https://github.com/vladich/pg_jitter)

Postgres JIT has been unusable for most queries since 2018 because LLVM takes tens to hundreds of milliseconds to compile. pg_jitter replaces it with three backends (sljit, AsmJit, MIR) that compile in tens of *microseconds* instead. The benchmarks are striking: LLVM sometimes runs 10-30x *slower* than the interpreter on wide tables, while pg_jitter's sljit backend is consistently 5-25% faster with near-instant compilation.

## When AI Writes the Software, Who Verifies It?

[Leo de Moura](https://leodemoura.github.io/blog/2026/02/28/when-ai-writes-the-worlds-software.html)

Leonardo de Moura (creator of Lean) makes the case for formal verification at AI scale. Google and Microsoft both report 25-30% of their code is now AI-generated, and Anthropic built a 100,000-line C compiler in two weeks for $20k. The problem: nearly half of AI-generated code fails basic security tests, and humans have stopped reading diffs carefully. His argument is that testing provides confidence but proof provides guarantees, and when AI generates code at thousand-times human speed, the verification gap widens into a systemic risk.

## Weave — Entity-level semantic merge for Git

[GitHub](https://github.com/Ataraxy-Labs/weave)

Weave replaces Git's line-based merge with entity-level merge using tree-sitter parsers. Instead of conflicting when two branches add different functions to the same file, it matches entities by identity (name + type + scope) and only conflicts on actual semantic collisions. The benchmarks show it resolving 31 of 31 test merges cleanly versus Git's 15 of 31. This matters most when multiple AI agents work on the same codebase—Git halts for human intervention on changes that are obviously independent.

## CFTC to Clear Path for U.S. Perpetual Futures

[CoinDesk](https://www.coindesk.com/policy/2026/03/03/cftc-chief-selig-to-clear-path-for-u-s-perpetual-futures-in-coming-weeks)

CFTC Chairman Selig announced the commission will approve perpetual futures for U.S. markets within weeks. Perps have been crypto's dominant trading instrument for years, but only offshore. Bringing them onshore changes the compliance calculus for every U.S. exchange and creates a legal path for institutional desks that have been waiting on regulatory clarity.

## AI Agents as Blockchain's Primary Users

[CoinDesk](https://www.coindesk.com/tech/2026/03/03/ai-agents-will-be-primary-users-of-blockchain-near-co-founder-says)

NEAR co-founder Illia Polosukhin argues AI will become the primary interface layer for everything online, including crypto, abstracting away wallets, explorers, and transaction hashes. The thesis: humans interact with AI, AI interacts with blockchains. If true, it inverts the entire UX stack—what matters is not making blockchains easier for humans to use, but making them easier for agents to access.

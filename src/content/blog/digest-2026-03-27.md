## Building a Digital Doorman with AI and IRC

**[George Larson](https://georgelarson.me/writing/2026-03-23-nullclaw-doorman/)**

George Larson built NullClaw, a public-facing AI agent running on a $7/month VPS that answers questions about his portfolio via IRC. The architecture splits security: a public-facing agent (NullClaw) handles visitor queries with read-only GitHub access, while a private agent (IronClaw) on a separate box handles email and calendar with no public exposure. Visitors interact through a web IRC client; NullClaw clones repos, reads code, and backs up claims with actual implementation details instead of resume platitudes. The agent uses Haiku for cheap conversational queries and Sonnet for tool-heavy work (repo cloning, code analysis). The A2A protocol handles agent-to-agent handoffs when NullClaw needs IronClaw's private context. This is the right way to deploy agents at perimeter: minimal attack surface, read-only tools, cost caps ($2/day), and zero private data on the public box.

## ATLAS Hits 74.6% LiveCodeBench with Frozen 14B Model

**[GitHub: itigges22/ATLAS](https://github.com/itigges22/ATLAS)**

ATLAS v3 achieved 74.6% pass@1-v (best-of-3 + iterative repair) on LiveCodeBench using a frozen, quantized Qwen3-14B model on a single RTX 5060 Ti 16GB. That's up from 36-41% in v2 and competitive with frontier API models at a fraction of the cost. The pipeline combines PlanSearch (constraint extraction + diverse solution generation), BudgetForcing (thinking token control), Geometric Lens (energy-based candidate scoring via 5120-dim self-embeddings), and PR-CoT repair (self-verified iterative refinement using model-generated test cases). ATLAS runs fully self-hosted—no API calls, no cloud, one GPU. The ablation study shows Phase 3 (self-verified repair) rescued 85.7% of failed tasks. This proves you can wrap a frozen smaller model in intelligent infrastructure (structured generation, verification, repair) and compete with frontier models for specific tasks like coding. The cost is latency (~20 min per hard problem vs. seconds for an API call), but when data can't leave the machine or API costs matter, this approach wins.

## jsongrep Outpaces jq with DFA-Based Search

**[Micah Kepe](https://micahkepe.com/blog/jsongrep/)**

jsongrep is a new JSON query tool that compiles queries into deterministic finite automata (DFAs) instead of interpreting path expressions at runtime. That means O(1) work per JSON node with no backtracking—queries are paid for once at compile time, then search is linear. Benchmarks show jsongrep crushing jq, jmespath, and jsonpath-rust on end-to-end search time, especially on large documents (~190MB). The tradeoff: higher upfront query compilation cost (DFA construction) and less expressive query language (no arithmetic, no transformations). But if you're filtering large JSON repeatedly or need predictable performance, compiling to DFA is the correct architecture. Written in Rust, zero-copy parsing via serde_json_borrow, cross-platform binaries available.

## Chroma Trains 20B Self-Editing Search Agent

**[Chroma Research](https://www.trychroma.com/research/context-1)**

Chroma released Context-1, a 20B parameter agentic search model trained on 8,000+ synthetic tasks to retrieve supporting documents for downstream reasoning models. The model operates as a retrieval subagent: it doesn't answer questions, it returns ranked documents. The key innovation is self-editing context—as the agent searches and its context window fills, it actively prunes irrelevant chunks to free space for further exploration. This prevents context rot (performance degradation from accumulated noise) and allows long-horizon search within a fixed token budget. Training uses a staged curriculum: first optimize for recall (find everything), then shift toward precision (keep only what matters). The reward function combines trajectory recall (credit for encountering relevant docs even if later pruned) and output F1 (balance recall/precision in final results). Context-1 matches frontier model retrieval performance at 10x inference speed and a fraction of the cost, running on a single B200 GPU with MXFP4 quantization at 400-500 tok/s. The full synthetic data generation pipeline and training code are open-sourced. This is the blueprint for building specialized retrieval agents that separate search from reasoning.

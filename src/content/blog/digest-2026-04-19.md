## AI & Dev Tools

**[Claude Opus 4.7](https://www.anthropic.com/news/claude-opus-4-7)** — Anthropic shipped Opus 4.7 with 13% gains on complex coding tasks and something unprecedented: runtime cyber safeguards that actively block offensive security requests. They deliberately reduced offensive cyber capabilities during training, then layered on automatic detection for prohibited uses. This is the first time I've seen a frontier lab differentially train down a dangerous capability, then gate what's left with real-time blocking. The new Cyber Verification Program lets legit pentesters bypass the safeguards. This is the blueprint for how they'll eventually ship Mythos-class models.

**[SmolVM](https://github.com/smol-machines/smolvm)** — Sub-200ms boot times for portable microVMs that run OCI containers (Docker images) with full hardware isolation. Each workload gets its own kernel on macOS Hypervisor.framework or Linux KVM, packed into a single `.smolmachine` file that runs anywhere the architecture matches. The project wraps libkrun into a CLI that sandboxes untrusted code without exposing host filesystem or network by default. SSH agent forwarding works without key exfiltration—private keys never touch the guest. Elastic memory via virtio balloon means the host only commits what the guest actually uses.

**[Claude Opus 4.6 → 4.7 system prompt changes](https://simonwillison.net/2026/Apr/18/opus-system-prompt/)** — Simon Willison diffed Anthropic's published system prompts and found telling shifts. The model now calls `tool_search` before claiming it lacks a capability (checking for deferred tools), asks fewer follow-up questions when the request is answerable, and won't beg you to stay when you say "bye." The child safety section tripled in size, and there's new language warning that once Claude refuses for child safety reasons, all subsequent messages in that conversation get extreme scrutiny. Anthropic's still the only major lab publishing these prompts.

**[B-52 Bomber Star Tracker](https://www.righto.com/2026/04/B-52-star-tracker-angle-computer.html)** — Before GPS, the B-52 used an electromechanical analog computer called the Angle Computer to navigate by stars. It physically models the celestial sphere with a mechanism that moves a pointer representing a star's position, then reads out azimuth and altitude electrically via synchros. Not a gyroscope, nothing spinning—just precision mechanics doing trigonometry for celestial navigation. Ken Shirriff tore one apart and documented the internals. Wild that this tech flew real missions.

**[Grok 4.3 beta](https://x.com/elonmusk/status/2045285099680252417)** — Elon dropped Grok 4.3 into early beta with natively multimodal capabilities. You can upload a screenshot of any website and Grok writes the code to clone it—no complex prompt needed. Front-end cloning via vision is table stakes now, but speed matters. X is using this to dogfood their AI moat and compete on inference latency. Still early beta, release notes coming as they fix bugs.

## Crypto & DeFi

**[Russia's Sberbank preps crypto trading](https://x.com/WhaleInsider/status/2045815393986519484)** — Russia's largest bank is rolling out crypto trading infrastructure. This follows their pivot after sanctions pushed them out of SWIFT. Banks that lose access to dollar rails start looking at alternatives. Expect more of this as fragmentation accelerates.

**[Hyperliquid's latency moat](https://x.com/MoziTenma/status/2045417647303799020)** — Stacy Muur pinged Hyperliquid from Tokyo (3ms) vs Amsterdam (221ms). That's physics: light travels through fiber at ~200 km/ms, and Hyperliquid's validators sit in Tokyo. At 100 book updates/second, Amsterdam traders are roughly 22 updates behind. Priority fees help, but the real moat is deterministic execution quality under stress. Geography plus infra spend becomes the edge. If you're trading perps on Hyperliquid from Europe, you're fighting latency before you fight the market.

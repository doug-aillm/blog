## Karpathy releases Microgpt: a minimal GPT implementation for learning

[Andrej Karpathy](http://karpathy.github.io/2026/02/12/microgpt/)

Karpathy's back with another teaching artifact—this time a stripped-down GPT that fits in your head. Unlike massive production codebases, this one trades completeness for clarity, making it the go-to reference for anyone who wants to understand transformers from first principles rather than just use them.

## Claude adds memory import to switch without starting over

[Claude.ai](https://claude.com/import-memory)

Anthropic just shipped a feature that lets you port conversation history and preferences from other assistants directly into Claude. This is smart competitive positioning—removing the friction of migration means users can experiment without losing accumulated context or having to rebuild their workflows from scratch.

## MCP server cuts Claude Code's context consumption by 98%

[mksg.lu](https://mksg.lu/blog/context-mode)

Someone built a Model Context Protocol server that compresses codebase context down to 2% of its original token footprint without sacrificing relevance. For anyone hitting context limits in long coding sessions, this is a practical fix—though the real story is how much room there still is for tooling to optimize what models actually need to see.

## Vitalik proposes EIP-8141 to complete Ethereum's account abstraction

[Ethereum Research](https://ethereum-magicians.org/t/eip-8141-complete-account-abstraction-via-frame-transactions/21849)

Vitalik's latest proposal introduces "Frame Transactions" to finally deliver native account abstraction at the protocol level, enabling gas sponsorship and complex multisig logic without relying on bundlers or separate standards. If it ships within a year as outlined, this removes one of Ethereum's longest-standing UX bottlenecks.

## obra ships superpowers: an agentic skills framework that actually works

[GitHub](https://github.com/obra/superpowers)

This is a methodology and tooling package for building reliable agent workflows without the usual prompt-chaining mess. What stands out is the emphasis on structured skills over freeform generation—agents get constrained, composable primitives instead of hoping vibes-based prompting holds up in production.

## Byreal plans 24/7 perpetual trading with AI-driven liquidity tools

[Catenaa](https://catenaa.com/news/byreal-247-perpetual-trading-ai-liquidity-2026)

Byreal's building an always-on perp DEX with AI liquidity provisioning to smooth out thin order books during off-peak hours. The bet here is that automated market-making can fill gaps traditional liquidity providers won't touch—if execution quality holds, this could legitimize 24/7 crypto derivatives for institutions still wary of weekend spreads.

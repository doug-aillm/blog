## Developer builds voice agent with sub-500ms latency from scratch

[Nick Tikhonov](https://www.ntik.me/posts/voice-agent)

A detailed breakdown of achieving sub-500ms end-to-end latency for voice AI agents. The trick is aggressive pipelining: streaming speech-to-text while generating responses and starting TTS before the full LLM output completes. Most commercial voice agents hit 1-2 second delays because they wait for each step to finish. This architecture proves you can cut that in half with careful buffering and partial result handling.

## Replit built a video rendering engine by lying to the browser about time

[Replit Blog](https://blog.replit.com/browsers-dont-want-to-be-cameras)

Browsers weren't designed as video rendering engines—`requestAnimationFrame` is locked to display refresh rates, making consistent video output impossible. Replit's solution: override browser time APIs to control frame timing precisely. They essentially trick the browser into thinking time passes at whatever rate they need for video export. It's a hack, but it works because browsers trust their own time APIs implicitly.

## Bank of Japan to test blockchain-based reserve settlement

[The Block](https://www.theblock.co/post/391945/bank-of-japan-blockchain-reserve-settlement)

Governor Ueda confirmed the BoJ will trial blockchain infrastructure for interbank settlement. This isn't about retail CBDCs—it's wholesale banking infrastructure targeting the same layer where SWIFT and correspondent banking operate today. Japan joins the ECB and Fed in exploring whether distributed ledgers can reduce settlement times and operational risk in the plumbing of the financial system.

## Programmable cryptography patterns from 0xPARC

[0xPARC](https://0xparc.org/writings/programmable-cryptography-1)

A deep dive into programmable cryptography—using ZK proofs, MPC, and FHE as composable building blocks rather than one-off implementations. The insight is treating cryptographic primitives like functions you can pipe together: prove computation correctness, then encrypt the result, then verify ownership, all without revealing underlying data. The examples show how this unlocks privacy-preserving apps that were structurally impossible before.

## Unicode explorer using binary search over HTTP range requests

[Simon Willison](https://tools.simonwillison.net/unicode-binary-search)

A clever demo that binary searches through a 76MB Unicode metadata file using HTTP range requests, never downloading the full file. Each search loads a few KB per step to find character information. The technique works because HTTP range requests let you grab arbitrary byte offsets, and Unicode codepoints sort naturally. It's a pattern that applies anywhere you have large sorted datasets—skip the download, search on-demand.

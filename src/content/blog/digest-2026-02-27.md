## Claude Code Builds Instead of Buying Tools

**[Amplifying AI Research](https://amplifying.ai/research/claude-code-picks)**

New research tracking 2,430 Claude Code responses across real repos shows it builds custom solutions 12 out of 20 times instead of recommending existing tools. When asked to add feature flags, it builds config systems with env vars and percentage rollout instead of suggesting LaunchDarkly. For Python auth, it writes JWT + bcrypt from scratch rather than reaching for libraries. When it does pick tools, the choices are decisive: GitHub Actions (94%), Stripe (91%), shadcn/ui (90%). The study also reveals sharp generational shifts—newer models favor newer tools, with Opus 4.6 completely replacing Prisma with Drizzle and dropping Celery for FastAPI BackgroundTasks. This shapes the default stack for thousands of projects built with AI assistance.

## SBI Holdings Launches Trust-Backed JPY Stablecoin

**[The Block](https://www.theblock.co/post/391386/sbi-holdings-startale-jpysc)**

Japan's SBI Holdings announced a trust bank-backed JPY stablecoin with a Q2 2026 launch target. Unlike typical reserve models, the stablecoin will be backed by assets held in a trust structure, which provides stronger legal segregation if the issuer faces insolvency. This follows Japan's stablecoin regulations that went live in 2023, requiring issuers to be banks or trust companies. The timing positions SBI ahead of major Japanese banks still evaluating their stablecoin strategies. If the trust model proves viable, expect other jurisdictions to adopt similar structures for regulatory clarity around reserve custody.

## Pipe Network Ships Free Solana Validator Client with Built-In CDN

**[Chainwire via The Block](https://www.theblock.co/press-releases/391445/pipe-network-launches-solanacdn-a-free-open-source-validator-client-with-built-in-acceleration-for-solana)**

Pipe Network released SolanaCDN, a free open-source validator client with integrated acceleration for Solana nodes. The client optimizes data distribution between validators, reducing bandwidth costs and improving sync times. Solana's high throughput (65K theoretical TPS) creates massive data replication requirements across validators—anything that cuts bandwidth or latency helps decentralization by lowering node operating costs. The open-source release means validators can audit the code before deploying, which matters for infrastructure that secures billions in assets.

## Understanding 80386 Protected Mode Memory Management

**[nand2mario](https://nand2mario.github.io/posts/2026/80386_protection/)**

Deep dive into the 80386's protected mode architecture, explaining segmentation, descriptor tables, and privilege levels that form the foundation of modern x86 memory protection. The article walks through how the CPU isolates processes using segment selectors and descriptor privilege levels, preventing user code from accessing kernel memory. This is the hardware mechanism that makes virtual memory and process isolation work on x86. The 80386 introduced these features in 1985, and they're still present (though largely unused in favor of paging) in every x86-64 CPU today. Worth reading if you're working on OS kernels, hypervisors, or low-level security tools.

## AirSnitch Breaks Wi-Fi Client Isolation

**[NDSS Symposium](https://www.ndss-symposium.org/wp-content/uploads/2026-f1282-paper.pdf)**

Security researchers published a method to break Wi-Fi client isolation, the feature that prevents devices on the same network from communicating directly. Client isolation is standard in hotels, airports, and cafes to stop lateral attacks between guests. AirSnitch exploits quirks in how access points handle broadcast frames and ARP caching to route traffic between isolated clients. The attack works against major AP vendors including Cisco, Ubiquiti, and Aruba. Practical impact depends on network config, but the research shows client isolation is weaker than assumed. If you manage public Wi-Fi, read the paper and check your AP firmware for patches.

## PersonaPlex 7B — Full-Duplex Speech on Apple Silicon

[Ivan's Blog](https://blog.ivan.digital/nvidia-personaplex-7b-on-apple-silicon-full-duplex-speech-to-speech-in-native-swift-with-mlx-0aa5276f2e23)

NVIDIA's PersonaPlex 7B runs full-duplex speech-to-speech on Apple Silicon using MLX in native Swift. The implementation achieves real-time voice interaction without round-tripping to cloud APIs, running entirely on-device with M-series chips. This matters because local speech models finally match cloud latency while keeping conversations private.

## Google Workspace CLI

[GitHub](https://github.com/googleworkspace/cli)

Google released an official CLI for Workspace that unifies Gmail, Drive, Calendar, and Docs into a single command-line interface. The tool supports scripting, bulk operations, and pipeline integration without OAuth dance for service accounts. For anyone managing multiple Workspace domains or automating admin tasks, this removes the need for custom API wrappers.

## Relicensing with AI-Assisted Rewrite

[tuananh.net](https://tuananh.net/2026/03/05/relicensing-with-ai-assisted-rewrite/)

A developer documented using Claude to rewrite an MIT-licensed library under Apache 2.0 by having the AI reimagine the implementation from specification. The legal premise: copyright protects expression, not ideas, so a functionally equivalent rewrite from requirements is clean-room. Whether this holds up in court is untested, but the workflow exists now and people are using it.

## SEC Submits Crypto Securities Framework to White House

[The Block](https://www.theblock.co/post/392337/sec-submits-framework-white-house)

The SEC delivered a formal framework to the White House on how securities laws should apply to crypto assets. This is the first time the commission has codified its position in a structured policy document rather than enforcement actions. If adopted, it sets the legal foundation for what counts as a security in crypto, ending years of regulation by litigation.

## The L in "LLM" Stands for Lying

[acko.net](https://acko.net/blog/the-l-in-llm-stands-for-lying/)

Steven Wittens argues that LLMs are structurally deceptive because they optimize for plausibility, not truth. The architecture rewards confidently wrong answers over admitting uncertainty, and RLHF trains models to avoid saying "I don't know" even when they should. The post walks through why this isn't a bug to be fixed but a property of next-token prediction—LLMs hallucinate because that's what the loss function incentivizes.
